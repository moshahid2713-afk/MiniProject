Project Prompt: 2D Terminal Graphics Editor

Build and improve the C terminal graphics editor in this project using the existing PDCurses-based setup.

Purpose
- Create and edit simple 2D shapes on a terminal canvas.
- Support keyboard and mouse interaction.
- Keep the interface simple, responsive, and easy to use in a text-mode environment.

Core requirements
1. Shape editing
- Support circle, rectangle, line, and triangle shapes.
- Allow shapes to be added, selected, modified, and removed.
- Keep the canvas and object list in sync.

2. Color handling
- Support multiple terminal colors for shape rendering.
- Allow the active color to be changed immediately with the c key.
- Ensure the current color is reflected in the status output and applied correctly to shapes on the canvas.

3. Fill mode and drawing
- Support filled and non-filled shapes where applicable.
- Render shapes correctly on the text canvas and preserve the existing visual style.

4. Zoom and layout
- Support zoom in and zoom out using + and -.
- Keep the canvas layout stable during resize and refresh.
- Ensure the menu, canvas, info, and status sections remain usable.

5. Mouse and keyboard interaction
- Allow mouse clicks to select shapes and create new shapes.
- Keep keyboard shortcuts simple and predictable: c for color cycle, + / - for zoom, r for refresh, and q for quit.
- Make sure status messages clearly tell the user what action was performed.

Implementation notes
- The main source file is graphics_editor.c.
- The project uses a local PDCurses dependency under deps/PDCurses.
- The code should compile with the provided build setup and remain stable in Windows terminal sessions.

Expected outcome
- The final program should behave as a polished terminal graphics editor with working shape creation, color changes, zoom controls, mouse interaction, and clear feedback to the user.
